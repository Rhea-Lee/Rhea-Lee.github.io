### **Overview**
***<u>S</u>ecure <u>SH</u>ell***

네트워크 상의 다른 컴퓨터에 안전하게 remote access 하기 위한 protocol이다.

단순히 접속만 하는 것이 아니라, 오가는 모든 데이터를 암호화하기 때문에 해킹으로부터 안전하다는 것이 가장 큰 특징이다.

<br>

### **Core Principle**
SSH는 기본적으로 **공개키(Public Key)**와 **개인키(Private Key)**라는 두 개의 열쇠를 사용하는 비대칭 암호화 방식을 사용한다.

- 공개키 (Public Key): 누구나 볼 수 있는 열쇠로, 접속하려는 원격 서버에 미리 저장함 (자물쇠 역할)
- 개인키 (Private Key): 나만 가지고 있는 비밀 열쇠로, 내 로컬 PC에 보관함 (진짜 열쇠 역할)

접속을 시도하면 서버가 "내가 가진 자물쇠(공개키)를 열어봐!"라고 요청하고, 내 PC가 올바른 개인키로 응답하면 로그인이 허용되는 구조이다.

<br>

### **Key Generation Algorithms**
- `RSA (Rivest-Shamir-Adleman)`: 가장 고전적이고 널리 쓰이는 알고리즘이다. 큰 정수의 소인수분해가 어렵다는 점을 이용한다. 보안을 위해 보통 3072비트 이상의 긴 키 길이를 권장한다.
- `ECDSA (Elliptic Curve Digital Signature Algorithm)`: 타원 곡선 수학을 이용한 현대적인 방식이다. RSA보다 훨씬 짧은 키 길이로도 동일한 보안 수준을 제공하며, 연산 속도가 빨라 CPU 리소스를 적게 소모한다.
- `Ed25519`: 최근 가장 권장되는 최신 알고리즘이다. 보안성이 매우 뛰어나고 키 생성이 빠르며, 키의 길이가 매우 짧아 관리가 용이하다는 장점이 있다.

<br>

### **Fingerprint & Hash Algorithms**
서버가 가진 긴 공개키를 식별하기 쉽게 짧은 문자열로 요약한 식별자로, 서버의 공개키 원본 데이터를 Hash Function(해시 함수)에 통과시켜 생성한 고정된 길이의 Hash Value(해시 값)이다.

- `SHA256 (Secure Hash Algorithm 256-bit)`: 현재 업계 표준으로 가장 널리 쓰이는 보안 해시 알고리즘이다. 보안성이 매우 강력하여 서로 다른 키가 같은 해시 값을 가질 확률이 사실상 제로에 가깝다. 보통 SHA256:Base64문자열 형태로 표현된다.
- `MD5 (Message-Digest Algorithm 5)`: 과거의 표준이었으나 현재는 보안 취약점이 발견되어 권장되지 않는 알고리즘이다. 주로 아주 오래된 구형 SSH 클라이언트와의 호환성을 위해 참고용으로만 출력되며, MD5:16진수 형태로 표현된다.

<br>

### **Connection & Authentication Steps**
사용자가 ssh 명령어를 입력하고 접속이 완료되기까지 내부적으로는 다음과 같은 단계를 거칩니다.

1. **Negotiation / Handshake & Key Exchange** <br>
  가장 먼저 클라이언트와 서버가 서로 지원하는 암호화 방식을 맞추고, 데이터를 암호화할 때 쓸 **대칭키**를 안전하게 교환한다.
  이 과정이 완료되면 이후 오가는 모든 데이터는 암호화된다.
2. **Server Authentication** <br>
  서버가 자신의 Public Key(Fingerprint)를 클라이언트에게 보낸다. 클라이언트는 이 지문이 내가 알던 서버가 맞는지 확인한다.
  - 처음 접속 시: Are you sure you want to continue connecting? 이라는 메시지가 뜨며, yes를 누르면 공개키 원본 데이터가 클라이언트의 ~/.ssh/known_hosts에 저장됨
  - 재접속 시: 저장된 지문과 서버가 보낸 지문을 자동으로 대조함
3. **User Authentication** <br>
  이제 서버가 접속하려는 사용자가 정당한 권한이 있는지 확인한다.
  - Password 방식: 암호화된 통로를 통해 사용자의 비밀번호를 전달받아 확인
  - Key 방식: 서버의 authorized_keys에 저장된 공개키와 사용자의 개인키를 대조하여 인증, 비밀번호를 매번 치지 않아도 되므로 보안과 편의성 면에서 더 우수함
4. **Session Established** <br>
  인증이 성공하면 암호화된 터널을 통해 원격 서버의 쉘(Shell) 환경에 접속하게 된다.

<br>

### **Usage Example**
```bash
# Basic SSH Connection
# Connect to the remote server using the default port (22)
ssh rhea@192.168.0.10

# Specifying a Custom Port
# -p (Port): Use a specific port if the server is not using the default port 22
ssh -p 2222 rhea@192.168.0.10

# Verbose Mode for Debugging
# -v (Verbose): Print debug messages to see what's happening during the connection
ssh -v rhea@192.168.0.10
```
